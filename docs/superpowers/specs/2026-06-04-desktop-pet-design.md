# Python 桌面宠物设计规格

日期：2026-06-04

## 目标

制作一个可在 Windows 上运行的 Python 桌面宠物第一版。程序启动后显示一个透明背景、始终置顶的动态宠物窗口。用户可以通过控制台命令随时切换宠物状态，也可以通过鼠标点击与宠物互动，并进行本地预设回复式对话。

第一版重点是跑通完整体验：置顶显示、动态效果、控制台控制、点击互动、本地对话。暂不接入在线 AI、语音、Live2D 或外部素材包，但代码结构会保留后续替换素材和扩展对话能力的空间。

## 技术选择

使用 Python 3.9 + PyQt5 + Pillow。

选择理由：

- 当前机器已安装 PyQt5 和 Pillow，可以直接开发运行。
- PyQt5 支持无边框窗口、透明背景、窗口置顶、鼠标事件和定时动画。
- Pillow 适合生成第一版内置宠物帧图，不需要额外图片资源。
- 单进程内使用后台线程读取控制台命令，降低首版复杂度。

## 用户体验

启动程序后，桌面上出现一只简单绘制的动态宠物。窗口没有边框，背景透明，并始终悬浮在其他应用上方。

宠物支持以下交互：

- 左键点击：触发动作变化，在 `happy`、`idle`、`walk`、`sleep` 等动作中按默认循环规则切换，不额外显示文字。
- 左键拖拽：移动宠物在桌面上的位置。
- 右键点击：弹出对话选择菜单，用户可以从菜单中选择预设对话主题。
- 控制台输入命令：立即切换宠物状态、让宠物说话、聊天、隐藏、显示或退出。

宠物显示的气泡文字会在一段时间后自动消失，避免长期遮挡桌面。

右键对话菜单第一版包含这些选项：

- `打招呼`：宠物用问候类回复回应。
- `问问状态`：宠物说出当前心情或状态。
- `陪我一下`：宠物给出陪伴类回复。
- `鼓励我`：宠物给出鼓励类回复。
- `随机聊天`：宠物随机选择一条本地回复。

## 第一版状态

宠物至少包含这些状态：

- `idle`：待机，身体轻微呼吸起伏，偶尔眨眼。
- `happy`：开心，身体轻跳，眼睛弯起。
- `sleep`：睡觉，眼睛闭合，显示睡眠气泡。
- `walk`：走动，身体左右摆动并缓慢横向移动。
- `talk`：说话，嘴部变化并显示气泡文字。
- `hidden`：窗口隐藏，但控制台仍可输入 `show` 恢复。

左键点击的动作变化规则：

- 如果宠物处于 `hidden` 以外的状态，左键点击会切换到下一个动作状态。
- 默认按 `happy`、`idle`、`walk`、`sleep`、`happy` 的顺序循环。
- 当宠物正在 `talk` 时，左键点击优先切换到 `happy`，避免文字气泡和动作切换冲突。
- 左键拖拽与左键点击分开处理：发生明显拖拽时只移动窗口，不触发动作切换。

## 控制台命令

控制台命令使用英文，便于输入和解析：

| 命令 | 行为 |
| --- | --- |
| `help` | 输出命令说明 |
| `idle` | 切换到待机 |
| `happy` | 切换到开心 |
| `sleep` | 切换到睡觉 |
| `walk` | 切换到走动 |
| `say <文本>` | 宠物显示指定文本并进入说话状态 |
| `chat <文本>` | 根据本地关键词规则回复 |
| `hide` | 隐藏宠物窗口 |
| `show` | 显示宠物窗口 |
| `quit` / `exit` | 安全退出程序 |

未知命令不会导致程序崩溃，而是在控制台提示可输入 `help` 查看说明。

## 本地对话

第一版使用本地预设回复，不需要联网，也不需要 API Key。

对话逻辑按关键词匹配：

- 问候类：`你好`、`hello`、`hi`
- 状态类：`你好吗`、`怎么样`
- 陪伴类：`无聊`、`陪我`
- 鼓励类：`累`、`压力`、`加油`
- 默认回复：未命中关键词时随机选择一条普通回应

对话回复会同时显示在控制台和宠物气泡中。

## 代码结构

计划的首版文件结构：

```text
desktop-pet/
  main.py
  pet_window.py
  pet_animator.py
  command_console.py
  dialogue.py
  README.md
```

模块职责：

- `main.py`：创建 Qt 应用，连接窗口、动画、控制台和对话模块。
- `pet_window.py`：管理透明置顶窗口、绘制图像、鼠标点击、拖拽、右键菜单、气泡文字和显示隐藏。
- `pet_animator.py`：管理宠物状态、动画计时、帧参数和状态切换。
- `command_console.py`：后台读取控制台输入，将命令转给主线程执行。
- `dialogue.py`：提供本地关键词回复。
- `README.md`：说明安装、运行、命令和后续替换素材方法。

## 线程和数据流

Qt 主线程负责所有 UI 操作。控制台读取运行在后台线程中，避免 `input()` 阻塞窗口动画。

数据流如下：

```text
控制台输入 -> CommandConsole -> Qt 信号 -> PetWindow / PetAnimator / LocalDialogue
鼠标左键点击 -> PetWindow -> PetAnimator
右键菜单选择 -> PetWindow -> LocalDialogue -> PetAnimator -> 气泡文字
动画定时器 -> PetAnimator -> PetWindow 重绘
```

后台线程不直接修改 UI，只通过 Qt 信号把命令传回主线程执行。

## 错误处理

- 未知命令：控制台输出提示，程序继续运行。
- 空命令：忽略。
- `say` 或 `chat` 缺少文本：提示正确用法。
- 窗口隐藏时输入动作命令：内部状态可更新，输入 `show` 后显示最新状态。
- 绘制异常：首版使用内置绘制逻辑，避免外部素材缺失导致启动失败。
- 退出：`quit` 和窗口关闭都应停止控制台线程并退出 Qt 应用。

## 测试和验收

手动验收：

- 程序可通过 `python main.py` 启动。
- 宠物窗口透明、无边框、始终置顶。
- 宠物可以被鼠标拖动。
- 左键点击宠物会触发动作变化。
- 左键拖拽宠物只移动位置，不误触发动作变化。
- 右键点击宠物会出现对话选择菜单。
- 右键菜单中的对话选项会让宠物显示对应回复。
- `idle`、`happy`、`sleep`、`walk`、`say`、`chat`、`hide`、`show`、`quit` 命令可用。
- 输入未知命令不会崩溃。

基础自动检查：

- 运行 Python 语法检查。
- 对 `dialogue.py` 的关键词回复做简单单元测试。
- 对命令解析函数做简单单元测试。
