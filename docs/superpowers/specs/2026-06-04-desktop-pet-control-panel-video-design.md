# 桌面宠物第二版控制面板与视频素材规格

日期：2026-06-04

## 目标

在第一版 Python 桌面宠物基础上新增一个可视化控制面板，减少对控制台命令的依赖。控制面板支持上传几秒钟的 MP4 宠物素材、控制宠物显示大小、切换动作、输入对话，并处理透明背景显示。

第二版素材策略是：优先使用视频帧自带的 Alpha 透明通道；如果当前 MP4 解码结果没有 Alpha 通道，则使用纯色背景透明化作为兜底。用户可以上传透明背景 MP4，也可以上传纯色背景 MP4。

## 保留内容

第一版已有能力继续保留：

- 桌面宠物窗口透明、无边框、始终置顶。
- 宠物窗口可左键拖拽。
- 左键点击宠物切换动作，不显示额外文字。
- 右键点击宠物显示对话选择菜单。
- 本地预设回复对话。
- 控制台命令保留为备用入口。
- 没有上传视频素材时，继续使用内置绘制宠物。

## 新增用户体验

程序启动后显示两个窗口：

- `PetWindow`：悬浮桌面宠物窗口，继续保持透明置顶。
- `ControlPanelWindow`：普通控制面板窗口，用来操作宠物。

控制面板包含这些区域：

- 素材区域：上传 MP4、恢复默认宠物、显示当前素材路径。
- 透明区域：显示透明模式、选择背景色、调整透明容差。
- 大小区域：通过滑块调整宠物显示大小。
- 动作区域：按钮控制 `idle`、`happy`、`sleep`、`walk`、`hide`、`show`。
- 对话区域：输入文字并触发 `say` 或 `chat`。
- 程序区域：退出程序。

控制台仍可使用，但用户日常操作优先通过控制面板完成。

## 视频素材处理

上传 MP4 后，程序读取视频帧并循环播放。视频素材应以几秒短循环为主，避免占用过多内存和 CPU。

视频帧处理规则：

- 如果读取到的帧包含 4 个通道，则把第 4 个通道作为 Alpha 透明通道。
- 如果读取到的帧只有 3 个通道，则进入纯色背景透明化模式。
- 纯色背景透明化模式会根据背景色和容差计算透明区域。
- 视频读取失败时，控制面板显示错误信息，并保持当前宠物素材不变。
- 用户点击“恢复默认宠物”后，停止视频播放并回到内置绘制宠物。

第一版透明背景兜底规则：

- 默认背景色为绿色 `#00FF00`。
- 可从绿色、白色、黑色中快速选择。
- 可打开颜色选择器指定自定义背景色。
- 透明容差滑块范围为 `0` 到 `120`，默认值为 `35`。
- 容差越大，被透明化的颜色范围越宽。

## 大小控制

宠物窗口大小由控制面板滑块控制：

- 范围：`50%` 到 `250%`。
- 默认：`100%`。
- 上传视频和内置绘制宠物都使用同一个大小控制。
- 修改大小时，宠物窗口应立即更新显示尺寸。
- 窗口仍可拖拽，缩放后不应跳回默认位置。

## 代码结构

在第一版文件基础上新增和调整：

```text
desktop-pet/
  main.py
  pet_window.py
  pet_animator.py
  pet_renderer.py
  command_console.py
  dialogue.py
  control_panel.py
  video_pet_source.py
  tests/
    test_video_pet_source.py
```

模块职责：

- `control_panel.py`：普通控制面板窗口，包含按钮、输入框、滑块、文件选择和颜色选择。
- `video_pet_source.py`：读取 MP4 视频帧，检测 Alpha 通道，执行纯色背景透明化，循环返回 `PIL.Image` 帧。
- `pet_window.py`：新增素材源和缩放控制；有视频素材时显示视频帧，没有视频素材时显示内置绘制宠物。
- `main.py`：创建宠物窗口和控制面板，连接控制面板信号到宠物窗口。
- `requirements.txt`：新增视频解码依赖，使用 `imageio` 与 `imageio-ffmpeg` 读取 MP4 帧。

## 数据流

```text
控制面板按钮 -> ControlPanelWindow -> PetWindow / PetAnimator / LocalDialogue
上传 MP4 -> ControlPanelWindow -> VideoPetSource -> PetWindow
大小滑块 -> ControlPanelWindow -> PetWindow.set_scale
透明设置 -> ControlPanelWindow -> VideoPetSource
动画定时器 -> PetWindow -> VideoPetSource 或 PetRenderer -> QLabel
控制台输入 -> CommandConsole -> PetWindow / PetAnimator / LocalDialogue
```

## 错误处理

- 用户取消文件选择：不改变当前素材。
- 选择非 MP4 文件：显示提示，不改变当前素材。
- 视频无法读取：显示提示，不改变当前素材。
- 视频帧尺寸过大：仍允许加载，但控制面板提示建议使用短视频和合理分辨率。
- Alpha 检测不到：自动使用纯色背景透明化，不中断播放。
- 透明容差设置过大导致主体被抠掉：用户可调低容差或恢复默认宠物。
- 大小滑块变化时如果视频素材不存在：内置绘制宠物正常缩放。

## 测试和验收

自动测试：

- `video_pet_source.py` 能识别 4 通道帧并保留 Alpha。
- `video_pet_source.py` 能对 3 通道纯色背景帧生成透明 Alpha。
- 透明容差变化会影响透明区域数量。
- 缩放值会被限制在 `0.5` 到 `2.5`。
- 没有视频素材时，宠物窗口仍使用内置绘制宠物。

手动验收：

- 运行 `python main.py` 后，宠物窗口和控制面板都出现。
- 控制面板按钮可切换宠物动作。
- 控制面板可输入文字让宠物说话或本地聊天。
- 上传 MP4 后，宠物窗口循环播放视频素材。
- 如果 MP4 带可读取 Alpha 通道，透明区域按 Alpha 显示。
- 如果 MP4 不带 Alpha，程序可用纯色背景透明化显示。
- 大小滑块可实时改变宠物显示尺寸。
- 恢复默认宠物后，回到内置绘制宠物。
- 控制台备用命令仍可使用。

## 非目标

第二版不做这些功能：

- 不接入在线 AI 对话。
- 不做语音识别或语音朗读。
- 不做复杂视频剪辑。
- 不保存多个素材库。
- 不打包成 Windows 可执行文件。

这些功能可以在后续版本继续扩展。
